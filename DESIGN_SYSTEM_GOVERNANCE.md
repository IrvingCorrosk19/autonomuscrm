# DESIGN_SYSTEM_GOVERNANCE

## Reglas definitivas RC
1. Uso obligatorio de clases `crm-*` en nuevas vistas y refactors.
2. Prohibido introducir estilos inline salvo excepciones documentadas.
3. Priorizar shared partials para modales, empty/loading y feedback.
4. Naming consistente para utilidades (`crm-*`) y componentes.
5. Mantener escala de spacing y densidad definida.
6. Estandarizar modales con overlay accesible y foco controlado.
7. Tablas con patrón responsivo + navegación keyboard.
8. Formularios con agrupación, ayuda contextual y feedback consistente.
